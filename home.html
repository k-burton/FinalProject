Lesson 1: Mapbox Studio

Think of Mapbox Studio as the Symbology tab in ArcGIS for desktop.

Despite having a very similar interface to that of arcGIS's home screen, Mapbox Studio is not the place where you watch your analysis as it's carried out; that happens in your browser. Based on my history with arcGIS, my very first assumption upon opening Mapbox Studio for the first time was that this was the central location for everything to happen. The layers of the map are visible in the left column, and it even shows what each layer's styling is in the thumbnail. I realized after some clicking around that this was not where you map from, but simply where you design your style.

This was the point at which I realized Mapbox was going to be a lot more unfamiliar than I had originally anticipated. I was able to complete the Choropleth Map Part 1 tutorial, shown in the above image, where I styled each division of population density data by hand. The process was reaally redundant compared to how it would be in ArcGIS. First, I had to make a new layer for each division of data (8 in this case) by copying the original layer. Then I had to give each layer its constraints (Less than 10, 10-20, 20-50, 50-100, 100-200, 200-500, 500-1000, 1000+).

The darkest color is for states with a population density of 1000 people / square mile or higher; New Jersey and Rhode Island. The next darkest color is for states with a population density between 500-1000 people per square mile; Massachusetts and Maryland.

Lesson 2: is code required?

Not knowing when a tutorial would require code made getting through the content difficult.

After easily completing the first part of the Choropleth Map tutorial, I continued on to part 2. However, there was no indication that I would need to code for this. The realization that much of Mapbox’s functionality was out of my reach came after a few more instances of failed tutorials.

Lesson 3: answering some questions

Beofore I could dive into the world of Mapbox code, I had to get answers to some important, basic questions...

What is a text editor?
How do html code and JavaScript go together?
Where do I see what's happening to my map?
How do you access geospatial tools? And are they ready to go?
This is where part 1 of the knowledge gap lies. Having taken a front end web development course and learning the basics of HTML CSS and JavaScript, I still haven’t bridged this part of the gap.

Lesson 4: Where to find the basics

Those basic analysis tools you’re looking for – buffer, measure, join, classify data, statistical functions, all live in the Mapbox Turf API

These were the first tools I went looking for, as they were the first things I learned in GIS courses. Even these most basic functions live in an API, and require programming.

Because Mapbox was built with the browser in mind, there are some capabilities that are superior to their counterparts on ArcGIS online.
